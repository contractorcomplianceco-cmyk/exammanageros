Core Record Structure:
One ExamManagerOS record per license/application.
A license/application may have multiple exams, usually two exams for the same person.

Default Owner:
Skylar should be the default owner for new exam records.

Record Creation Flow:
When an invoice is paid in Zoho Books and includes the Exam Scheduling item number, create an exam record and route it to QA.

Represent this in the prototype as:
- Integration status card
- “Zoho Books trigger: Invoice Paid”
- “Exam Scheduling item detected”
- “Record created”
- “QA queue created”
- “Default owner: Skylar”

Required Integrations To Represent:
1. Zoho Books
   - Paid invoice triggers exam workflow.
   - Item number for “Exam Scheduling” will be created by Rose.
   - Prototype must show this as a placeholder integration only.

2. AuditEngine
   - Separate source connected later.
   - Provides exam requirements, pre-approval rules, vendors, costs, deadlines, passing requirements, retest rules, pass sheets, and state/statute rules.
   - Prototype must show an AuditEngine Rules panel.

3. ComplianceConnect
   - Client-facing portal shows basic exam information, scheduler contact, status, next steps, timeline, vendor, location, date/time, and prep status.
   - Do not show internal notes.
   - Prototype must include a client preview.

4. DocCollect
   - Clients upload pass sheets, exam confirmations, approvals, and related documents through ComplianceConnect/DocCollect.
   - ExamManagerOS and DocCollect should share access to the document folder.
   - Pass sheet is required before completion.
   - When pass sheet is uploaded, notify Doc Collection/Application Processing.
   - Prototype must include document status panels.

Scheduling Workflow:
If no pre-approval is needed:
1. Send client exam information.
2. Send exam prep information if purchased.
3. Request client testing date preferences.
4. Book the exam.
5. Send scheduled exam confirmation.
6. Get pass sheet.
7. Update Doc Collection/Application Processing.

1. Send client exam information.
2. Wait for board approval.
3. Once approved, request client testing date preferences.
4. Book the exam.
5. Send scheduled exam confirmation.
6. Get pass sheet.
7. Update Doc Collection/Application Processing.
